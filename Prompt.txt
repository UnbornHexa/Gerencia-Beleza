O objetivo deste programa é desenvolver uma aplicação full-stack moderna que integre múltiplas linguages e serviços com foco em integração rápida entre sistemas e deploy facilitado para desktop e mobile app. A aplicação é uma plataforma de gerenciamento de sistema de beleza para funcionários autonomos (i.e Manicures à domicílio).

Você deverá construir um sistema que:

1. Tem uma tela de login aonde os usuários precisam logar antes de serem levados à própria Dashboard deles. E nesta tela de login também ter a opção para criar uma conta.

2. Uma tela de Criação de conta conectada ao entrar que faça a adição da conta nova ou informa que a conta já existe se o email já estiver registrado. Queremos coletar da profissional que está se registrando o email (chave principal), a senha que ela deseja, o endereço de onde mora dividido por input de CEP, estado, cidade, rua, numero e complemento(Com integração da API IBGE para facilitar encontrar endereço pelo CEP e para escolher estado e cidade) e o telefone para contato.

3. Um CRUD para os clientes da profissional que usa o programa.

4. Um CRUD para os serviços que a profissional providencia incluindo informações básicas de preço, nome do serviço e observação sobre ele.

5. Uma tela que demonstre e registre ganhos e despesas:
- Nesta tela as informações dos ganhos e gastos são demonstrados, normalmente os desse mês, mas filtros permitem checar datas específicas ou períodos específicos com os filtros mais comuns e acessíveis sendo "mês" "3 meses" "6 meses" e "Este ano".
- Gráficos que mostrem quanto foram os ganhos e despesas totais dentro do filtro de tempo assim como dividir pela classificação dos ditos entradas e saídas. Os diferenciais de entrada são pelo nome do serviço que está registrado de forma que a profissional pode saber qual serviço que ela disponibiliza está rendendo mais. Os gastos devem ser divididos nas categorias Saúde, Educação, Alimentação, Gastos Pessoais, Trabalho, Veículo, Lazer, Casa.
- Nesta tela também devem haver botões de Adicionar Ganho e Adicionar Despesa que abrem um modal para registrar o dito ganho ou despesa. Ele exige nome, valor e no caso das despesas pede para escolher uma das tags que foram enumeradas acima.
- Permite exportação do relatório financeiro do filtro em questão (Mês em questão, ano em questão, entre data x e y) em PDF com detalhamento e formatação de página para fácil leitura e compreensão dos valores, proporções e fluxo de caixa do período.

6. O backend aonde armazena os dados é uma API NestJS com MongoDB

7. Uma das telas precisa ser a agenda, essa agenda precisa de filtro para mostrar os compromissos do dia, por semana ou pelo mês de modo que muda a visualização do calendário. Baseie-se no Google Calendários ou até mesmo integre com ele se achar necessário. Aqui ficam os horários das clientes da profissional e o que foi marcado.
- Se você for criar uma própria nossa, tenha certeza de integrar com a tela de serviços para que ao criar um horário e a profissional vá escolher os serviços eles são carregados num dropbox com a lista de serviços dela e que mais de um serviços pode ser adicionado no horário. Também é bom colocar nesse agendamento da cliente o total do valor que a profissional vai ganhar se fizer esse agendamento. Este total deve ser exibido tanto na hora que os serviços estão sendo adicionados quanto num pequeno número no card do horário em questão. Qualquer horário que vá ser deletado precisa ser primeiro confirmado por um modal que também pede para a profissional colocar a razão da deleção ou cancelamento do horário, colocar abaixo do input um pequeno lembrete de que é importante preencher isso para ajudar a alimentar o programa com os hábitos de suas clientes.

8. Exibe um Dashboard no frontend (React + Vite + Tailwind + shadcn/ui) aonde são exibidos:
- O valor total de caixa (ganhos - despesas) do mês anterior, do mês atual e do mês seguinte (Caso ajam despesas ou ganhos programados para o mês seguinte) com porcentagem abaixo deles mostrando em quantos % foi o aumento ou baixa comparado ao mês anterior ou seguinte respectivamente.
- Também deve ser mostrado nessa parte uma projeção do quanto a profissional ainda tem para ganhar neste dia segundo os horários que tem programados para a agenda daquele dia de forma a inspirar a profissional a seguir o dia.
- O botão Adicionar Gasto e Adicionar Despesa devem estar presentes aqui também para facilidade de acesso da profissional.
- Abaixo destes devemos exibir o gráfico de ganhos e gastos que foi feito na página de finanças, para fácil acompanhamento financeira ao entrar no sistema.
- Além disso um local dedicado na dashboard para mostrar as próximas clientes que a profissional tem agendada para as próximas 3 horas para que possa se programar e quando clicar no agendamento mostrar um modal com os detalhes do atendimento em questão. Assim como colocar um botão ou texto pequeno nesse pedaço que leve direto para a agenda, algo como "Ir para agenda" para caso a profissional queira ir consultar a agenda direto dali.
- No modal que mostra o agendamento da cliente que vem a seguir também deve ter um botão para Confirmar, Remarcar ou Cancelar. Quando um destes botões é apertado ele primeiro pede confirmação do usuário antes de seguir com a ação. Aqui você deve integrar a API do Whatsapp para que o clique use o Whatsapp da profisisonal(Registrado na hora de criar a conta) para criar uma mensagem no chat com a cliente (numero registrado no CRUD de cliente) de confirmação de que o horário vai ser cumprido, de que ele vai ter que ser cancelado ou então que terá que ser remarcado por causa de um atraso, mas não envie, deixe em rascunho e leve a profissional para a tela do chat de forma que ela possa editar algo se desejar antes de enviar.

9. Tela de Meu Perfil para a profissional poder editar informações como o número de telefone caso tenha mudado, assim como o email dela e até mesmo trocar a senha caso deseje. Além disso aqui é onde são configuradas as mensagens padrões para o Whatsapp. Precisa ter uma mensagem para quando for Confirmar, uma para quando tiver que Remarcar e uma quanto for Cancelar.

10. Toda a solução deve rodar via Docker Compose e deve funcionar localmente assim como ao subir para deployment que será feito na Hostinger.

11. Gera insights baseados em IA que podem ser criados por você mesmo ou integrados com uma IA externa free que você encontre. Os insights são coisas como:
- Mostrar no perfil de cada cliente quais os serviços que ela mais contrata normalmente. Mostrar como qual horário de atendimento (manhã, tarde ou noite) a cliente prefere e até mesmo o horário preferido.
- Detectar padrões de quando clientes marcam horário com a profissional (semanalmente, quinzenalmente, mensalmente) e criar lembretes na dashboard com relação a isso, por exemplo: Fulana costuma fazer unha quinzenalmente e faz 2 semanas sem, mande uma mensagem para lembra-la. Junto destes lembretes deve ter um botão que é Enviar lembrete que ativa a integração com o Whatsapp API para enviar uma mensagem para a cliente em questão perguntando se ela gostaria de fazer em tal dia e hora, já colocar no rascunho de mensagem o serviço que normalmente é feito assim como sugerir o horário que a cliente normalmente prefere. Como em outros recursos, a mensagem deve ser um rascunho e a profissional é levada para a tela da conversa do whats para adaptar como ela quiser antes de enviar.
- Analizar e concluir quais os serviços que mais são contratados e mostrar na tela de finanças dentro de um card sobre Análise de Serviços que também fala sobre quantas vezes foram contratados esses serviços no ultimo mês e por quantas clientes ele foi contratado. Criar um top 3 de serviços que se mantém atualizado a qualquer momento.
- Mostrar também qual bairro normalmente traz o maior rendimento de dinheiro para a profissional, mostrar o top 3 dos bairros.
- Essas informações de top 3 devem sempre poder ser filtradas por top 3 do mês, top 3 do semestre e top 3 do ano.
- Adicionar e mostrar uma anotação nas clientes que gastem bastante por mês para que a profissional possa saber que essas são prioridades. Essa anotação também deve ser mostrada na dashboard aonde fala os horários que virão, se uma cliente VIP estiver marcada deve ficar claro isso para a profissional ver.

Stack obrigatória
Frontend: React + Vite + Tailwind + shadcn/ui
Backend (API): NestJS (TypeScript)
Banco de dados: MongoDB (Atlas ou container)
APIs Externas:
- Whatsapp API
- Google Calendar
- IBGE API
Infra: Docker / Docker Compose
Linguagem base: TypeScript obrigatório (frontend e backend)

API (NestJS + MongoDB)
A API em NestJS será o núcleo do sistema, responsável por:

Receber e armazenar os dados;
Expor endpoints para consumo pelo frontend;
Orquestrar ou acionar a camada de IA;
Gerenciar clientes e serviços.
Expor endpoints

Critérios
Clareza de arquitetura: organização de pastas, camadas e responsabilidades;
Qualidade de código: tipagem, legibilidade, padrões adotados;
Integração entre serviços: comunicação estável e bem tratada;
Boas práticas: validação, tratamento de erros, logs, eslint/prettier;
UX: experiência de uso do Dashboard e das telas;
Criatividade: na forma de mostrar dados e insights;
Documentação: README claro, com passos de execução e configuração;
Uso correto do Docker Compose para subir tudo.
Logs detalhados por serviço;
CI (lint/test) configurado;
Levar em consideração de que esse aplicativo será também usado em celular e que um app será desenvolvido para ele baseado nesse código.

REGRAS
Não coletar ou armazenar dados pessoais sensíveis;
Usar intervalos razoáveis para chamadas às APIs externas;
Focar em integração, clareza e coesão, não apenas em adicionar complexidade;
Você é livre para:
Renomear endpoints;
Alterar nomes de coleções;
Mudar estruturas de diretórios;
Escolher bibliotecas auxiliares
